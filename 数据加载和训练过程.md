## 第一部分：DataLoader 的数据结构与使用方式

### 数据从哪里来？

你的数据集是 **Tiny ImageNet-200**：200个类别，每张图片是 64×64 像素的彩色图。

```
dataset-tiny-imagenet-200/
  train/
    n01443537/   ← 类别1（金鱼）
      image1.jpg
      image2.jpg
      ...
    n01629819/   ← 类别2（蜥蜴）
      ...
  val/
    ...
```

PyTorch 的 `datasets.ImageFolder` 会自动读取这个文件夹结构，把 **文件夹名当作类别标签**。

---

### 数据预处理（Transform）

原始图片是 0~255 的像素值，要先处理成神经网络能用的格式：

```52:54:tiny_imagenet/data_loader.py
    train_dataset = datasets.ImageFolder(root=root / "train", transform=train_transform)
    valid_dataset = datasets.ImageFolder(root=root / "train", transform=_EVAL_TRANSFORM)
```

训练集的 `train_transform` 会做四件事：

```python
transforms.RandomCrop(64, padding=4),   # 先在边缘补4像素，再随机裁回64×64（数据增强）
transforms.RandomHorizontalFlip(),       # 50%概率水平翻转（数据增强）
transforms.ToTensor(),                   # 图片 → 张量，像素值从[0,255]变成[0.0,1.0]
_NORMALIZE,                              # 标准化：减均值、除以标准差，让数值分布在0附近
```

---

### DataLoader 长什么样？

`train_data_loader` 和 `valid_data_loader` 是 PyTorch 的 `DataLoader` 对象，可以把它理解成一个**自动打包机器**：

- 每次从数据集里取出 `batch_size` 张图片
- 把它们打包成一个 **batch（批次）**
- 自动随机打乱顺序

当你在训练循环中写：

```python
for images, labels in train_loader:
```

每次迭代，你拿到的是：

| 变量     | 形状              | 含义                                      |
| -------- | ----------------- | ----------------------------------------- |
| `images` | `(64, 3, 64, 64)` | 64张图，每张3通道（RGB），每通道64×64像素 |
| `labels` | `(64,)`           | 64个整数，对应每张图的类别编号（0~199）   |

---

### 训练集 vs 验证集 的区别

这里有个关键细节：两个 loader **都来自同一个 train 文件夹**，但通过 `SubsetRandomSampler` 切割：

```55:66:tiny_imagenet/data_loader.py
    indices = list(range(len(train_dataset)))
    split = int(np.floor(valid_size * len(train_dataset)))
    if shuffle:
        np.random.seed(random_seed)
        np.random.shuffle(indices)

    train_loader = _make_loader(
        train_dataset, batch_size, sampler=SubsetRandomSampler(indices[split:])
    )
    valid_loader = _make_loader(
        valid_dataset, batch_size, sampler=SubsetRandomSampler(indices[:split])
    )
```

- 总共 100,000 张训练图，按 `valid_size=0.1` 分割
- **train_loader**：拿后 90%（90,000 张）用来训练
- **valid_loader**：拿前 10%（10,000 张）用来验证，**不参与训练**
- 验证集不做数据增强（不 crop、不 flip），保证评估公平

## 第二部分：Training 训练过程

训练的核心目标：**让模型预测的结果越来越准确**。

这通过一个叫做**反向传播（Backpropagation）**的算法实现，每一步训练分4个动作：

```
前向传播 → 计算损失 → 反向传播 → 更新参数
```

---

### 整体结构：两层循环

```11:51:COMP5541_AlexNet_TinyImageNet.ipynb
def fit(model, train_loader, test_loader, criterion, optimizer, num_epochs=30):
    for epoch in range(num_epochs):          # ← 外层循环：重复30遍
        model.train()
        for i, (images, labels) in enumerate(train_loader):  # ← 内层循环：每次处理一个batch
```

- **外层 epoch 循环**：完整过一遍所有训练数据叫做 1 个 epoch，共跑 30 次
- **内层 batch 循环**：90,000 张图 ÷ 64（batch size）≈ 1407 步，所以你看到日志里 `Step [1407/1407]`

---

### 每一步（Step）发生了什么？

以第 `i` 步为例，手里有 64 张图片和对应标签：

**① 前向传播（Forward Pass）**

```python
outputs = model(images)   # 把64张图喂给AlexNet，得到 (64, 200) 的预测结果
                          # 每张图对200个类别各有一个"得分"
```

**② 计算损失（Loss）**

```python
loss = criterion(outputs, labels)   # 用CrossEntropyLoss衡量"预测得分"和"真实标签"的差距
                                    # 差距越大，loss越大
```

**③ 清零梯度**

```python
optimizer.zero_grad()   # 把上一步留下的梯度清零，否则会累加
```

**④ 反向传播（Backward Pass）**

```python
loss.backward()   # 自动计算：loss对每个参数的"偏导数"（梯度）
                  # 梯度告诉我们：调整哪个参数、往哪个方向调，能让loss下降
```

**⑤ 更新参数（Optimizer Step）**

```python
optimizer.step()   # SGD优化器按照梯度方向，把所有参数微调一点点
                   # learning_rate=0.01 决定每步迈多大
```

---

### 一个直觉类比

> 想象你在黑暗中下山（loss 是海拔高度），每一步你用手摸地面的坡度（反向传播计算梯度），然后往坡度最陡的下坡方向迈一小步（optimizer.step 更新参数）。重复 30 × 1407 = 42,210 次，终于走到山脚（loss 最小，准确率最高）。

---

### 每个 epoch 结束后

```python
train_loss = total_loss / total      # 这个epoch平均每张图的loss
train_acc  = 100.0 * correct / total # 这个epoch训练集准确率

test_loss, test_acc = evaluate(model, test_loader, criterion)  # 在验证集上测一次，看泛化能力
```

`evaluate` 函数里有 `torch.no_grad()`：验证时**不计算梯度**、不更新参数，只是纯粹看看预测准不准。
